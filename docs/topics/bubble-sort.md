# バブルソート

## 1. 概要

バブルソートは、隣り合う要素を比較し、逆順なら交換する処理を繰り返して並べ替えるソートアルゴリズム。

1回の走査が終わるごとに、大きい値が右端へ押し出されていく。  
処理の流れが見やすく、trace と相性がよい基本ソートのひとつ。

---

## 2. 基本の動き

バブルソートでは、左から右へ順に隣接要素を比較する。

- 左の値が右の値より大きい場合は交換する
- これを未整列区間の末尾まで繰り返す
- 1周すると、その区間の最大値が右端に確定する
- 次の周では、右端の確定済み要素を除いて同じことを行う

「大きい値が泡のように右へ浮き上がる」という説明がよく使われる。

---

## 3. 処理の流れ

処理の流れは次の通り。

1. 配列の末尾を未確定区間の終端として設定する
2. 先頭から順に隣接要素を比較する
3. 逆順なら交換する
4. 1周終わったら、その区間の最大値が右端に確定する
5. 未確定区間を1つ縮めて再度同じ処理を行う
6. 交換が1回も起きなければ、整列済みと判断して終了する

---

## 4. 計算量

- 時間計算量
  - 最悪: O(n^2)
  - 平均: O(n^2)
  - 最良: O(n)  
    交換が発生しなかった時点で終了する実装の場合

- 空間計算量
  - O(1)  
    配列内で交換しながら並べ替えるため

---

## 5. メリット / デメリット

### メリット

- 処理の流れが単純で理解しやすい
- 隣接比較と交換という基本操作で構成されている
- trace で途中経過を追いやすい

### デメリット

- 基本的に O(n^2) で遅い
- 配列サイズが大きくなると比較回数が急増する
- 実務でそのまま使う場面は少ない

---

## 6. Java実装のポイント

- 配列をそのまま壊さず比較できるよう、最初に `Arrays.copyOf` で複製している
- 外側のループで未整列区間の終端を縮めている
- 内側のループで隣接要素を比較している
- `swapped` を使って、交換が起きなかった場合は早期終了している
- 比較回数、交換回数、参考実行時間を結果として返している

---

## 7. コアとなる処理・重要なコードの見どころ

最も重要なのは、隣接要素を比較して必要なら交換する内側ループ。

```java
for (int i = 0; i < end; i++) {
    comparisonCount++;

    if (values[i] > values[i + 1]) {
        int temp = values[i];
        values[i] = values[i + 1];
        values[i + 1] = temp;
        swapCount++;
        swapped = true;
    }
}
```

ここで、

- `values[i] > values[i + 1]` を比較する
- 逆順なら交換する
- 1周の中で最大値が右端へ移動する

という、バブルソートの本体処理を実現している。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic bubble-sort --trace
```

### 入力指定

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic bubble-sort --input 5,3,8,1,4 --trace
```

### 大きい入力

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic bubble-sort --size 1000
```

実行時は、入力配列、途中経過、ソート後配列、比較回数、交換回数、参考実行時間を確認できる。
