# 分割統治

## 1. 概要

分割統治は、問題を小さな部分問題に分け、それぞれを解いた後に結果を統合する考え方。  
再帰と組み合わせて使うことが多く、後続のマージソートやクイックソートの土台になる。

今回のトピックでは、分割統治の基本を確認するために、次の題材を扱う。

- 配列合計
- 配列最大値

---

## 2. 基本の動き

分割統治では、次の流れで処理を進める。

- 問題を左右 2 つなどの小さい部分問題に分ける
- それぞれを再帰的に解く
- 最後に左右の結果を統合する

この「分ける」「解く」「統合する」の 3 段階が中心になる。

---

## 3. 処理の流れ

処理の流れは次の通り。

1. 対象区間が要素 1 つかどうかを確認する
2. 要素 1 つなら、その値をそのまま返す
3. 要素が複数あるなら、中央で左右 2 つの区間に分ける
4. 左右それぞれを再帰的に解く
5. 左右の結果を足し合わせる、または比較して統合する
6. 統合結果を呼び出し元へ返す

trace を有効にすると、区間の分割と統合の流れを確認できる。

---

## 4. 計算量

今回のトピックで扱う配列合計と最大値探索では、各要素を 1 回ずつ使って左右に分けながら計算するため、時間計算量は O(n) になる。

- 配列合計: O(n)
- 配列最大値: O(n)

空間計算量は、再帰呼び出しの深さに応じて O(log n) 程度になる。

---

## 5. メリット / デメリット

### メリット

- 大きい問題を小さく分けて考えやすい
- 再帰との相性がよい
- 後続の高速ソートや木・グラフ系の考え方につながる

### デメリット

- 分割と統合の流れを追う必要がある
- 単純なループより実装が長くなりやすい
- 小さい問題ではループより分かりにくいことがある

---

## 6. Java実装のポイント

- 入力配列は `Arrays.copyOf` で複製し、元配列を壊さずに学習できるようにしている
- `left` と `right` で現在の対象区間を管理している
- `mid` を計算して左右に分けている
- 呼び出し回数を表示するために補助カウンタを使っている
- trace を有効にすると、区間分割と統合結果を表示する

---

## 7. コアとなる処理・重要なコードの見どころ

最も重要なのは、区間を半分に分けて左右を再帰呼び出しし、最後に統合する部分。

```java
int mid = left + (right - left) / 2;

long leftValue = sumRecursive(values, left, mid, counter, trace);
long rightValue = sumRecursive(values, mid + 1, right, counter, trace);
long result = leftValue + rightValue;
```

ここで、

- `mid` で区間を分割する
- 左右の部分問題をそれぞれ解く
- 最後に左右の結果を統合する

という、分割統治の本体処理を確認できる。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic divide-and-conquer --trace
```

### 入力指定

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic divide-and-conquer --input 8,3,7,4,9,2,6,5 --trace
```

### 大きい入力

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic divide-and-conquer --size 1000
```

実行時は、配列合計と最大値探索を通して、区間分割と統合の流れを確認できる。
