# 再帰

## 1. 概要

再帰は、メソッドの中から自分自身を呼び出して処理を進める考え方。  
問題を少しずつ小さくし、最後にこれ以上分けられない状態に到達したら処理を止める。

今回のトピックでは、再帰の基本を確認するために、次の題材を扱う。

- カウントダウン
- 階乗
- 1 から n までの合計

---

## 2. 基本の動き

再帰では、主に次の 2 つが重要になる。

- **ベースケース**  
  これ以上自分自身を呼ばず、処理を終了する条件

- **再帰ケース**  
  問題を少し小さくして、自分自身を再度呼ぶ処理

この 2 つがそろうことで、処理を止めながら結果を組み立てられる。

---

## 3. 処理の流れ

処理の流れは次の通り。

1. 現在の入力がベースケースかどうかを確認する
2. ベースケースなら、その場で結果を返して処理を終了する
3. ベースケースでなければ、より小さい問題で自分自身を呼ぶ
4. 呼び出し先から結果が返ってきたら、その結果を使って現在の結果を作る
5. その結果を呼び出し元へ返す

trace を有効にすると「入る」「ベースケース」「戻りがけの計算」の流れを確認できる。

---

## 4. 計算量

今回のトピックで扱う再帰の例では、基本的に入力値 n に対して 1 回ずつ呼び出しが増えるため、時間計算量は O(n) になる。

- カウントダウン: O(n)
- 階乗: O(n)
- 1 から n までの合計: O(n)

空間計算量も、呼び出しが積み重なるため O(n) になる。

---

## 5. メリット / デメリット

### メリット

- 問題を小さくして考える流れが明確になる
- 木構造や分割統治など、後続トピックにつながりやすい
- 処理の定義をそのままコードにしやすい

### デメリット

- 呼び出しの流れを頭の中で追う必要があり、最初は分かりづらい
- ベースケースを誤ると無限再帰になる
- ループより呼び出しコストが増えることがある

---

## 6. Java実装のポイント

- 呼び出し回数を表示するために補助カウンタを使っている
- trace を有効にすると、呼び出し開始と戻りの流れを表示する
- `factorial` と `sumFromOneToN` は、戻りがけに値を組み立てる形にしている
- `countDown` は、ベースケース到達まで自分を呼ぶ流れを見るための題材として使っている

---

## 7. コアとなる処理・重要なコードの見どころ

最も重要なのは、ベースケースと再帰呼び出しが並んでいる部分。

```java
if (n <= 1) {
    return 1;
}

long subResult = factorialRecursive(n - 1, counter, trace);
long result = n * subResult;
return result;
```

ここで、

- `n <= 1` がベースケース
- `factorialRecursive(n - 1, ...)` が再帰呼び出し
- 戻ってきた結果を使って現在の結果を作る

という、再帰の基本形を確認できる。

---

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic recursion --trace
```

### 入力指定

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic recursion --input 6 --trace
```

### 大きい入力

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic recursion --size 10
```

実行時は、カウントダウン、階乗、1 から n までの合計を通して、再帰呼び出しと戻りがけの流れを確認できる。
