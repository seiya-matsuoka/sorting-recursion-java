# 選択ソート

## 1. 概要

選択ソートは、未整列区間から最小値を探し、その値を先頭へ移動させることを繰り返して並べ替えるソートアルゴリズム。

毎回「次に確定させる最小値」を選ぶため、比較の流れが見やすい基本ソートのひとつ。

---

## 2. 基本の動き

選択ソートでは、配列の左側から順に整列済み区間を広げていく。

- まず未整列区間全体から最小値を探す
- 見つかった最小値を、その区間の先頭と交換する
- これで左端の1要素が確定する
- 次はその右隣から同じことを行う

「最小値を選んで前へ持ってくる」という考え方が中心になる。

---

## 3. 処理の流れ

処理の流れは次の通り。

1. 左端を、今回確定させる位置とする
2. その位置以降を未整列区間として扱う
3. 未整列区間を走査して最小値の位置を探す
4. 見つけた最小値を先頭位置と交換する
5. 確定位置を1つ右へ進める
6. 配列の末尾近くまで同じ処理を繰り返す

---

## 4. 計算量

- 時間計算量
  - 最悪: O(n^2)
  - 平均: O(n^2)
  - 最良: O(n^2)

- 空間計算量
  - O(1)

比較回数は入力の並びにかかわらず大きくは減らない点が特徴。

---

## 5. メリット / デメリット

### メリット

- 最小値を探して確定させる流れが分かりやすい
- 交換回数は比較的少なくなりやすい
- trace で最小値候補の更新が見やすい

### デメリット

- 比較回数は多く、O(n^2) になる
- 配列サイズが大きくなると遅い
- 実務でそのまま使う場面は少ない

---

## 6. Java実装のポイント

- `left` より左側を整列済み区間として扱っている
- `minIndex` を使って最小値候補の位置を保持している
- 内側ループで未整列区間を最後まで走査している
- 最小値が先頭にない場合だけ交換している
- 比較回数、交換回数、参考実行時間を結果として返している

---

## 7. コアとなる処理・重要なコードの見どころ

最も重要なのは、未整列区間から最小値の位置を探す部分。

```java
for (int i = left + 1; i < values.length; i++) {
    comparisonCount++;

    if (values[i] < values[minIndex]) {
        minIndex = i;
    }
}
```

ここで、

- 未整列区間を順に見る
- より小さい値が見つかったら最小値候補を更新する
- 最後にその位置を先頭と交換する

という、選択ソートの中心処理を実現している。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic selection-sort --trace
```

### 入力指定

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic selection-sort --input 5,3,8,1,4 --trace
```

### 大きい入力

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic selection-sort --size 1000
```

実行時は、入力配列、最小値候補の更新、ソート後配列、比較回数、交換回数、参考実行時間を確認できる。
