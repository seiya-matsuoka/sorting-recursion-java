# 挿入ソート

## 1. 概要

挿入ソートは、左側の整列済み区間に対して、現在の値を適切な位置へ挿し込むことを繰り返して並べ替えるソートアルゴリズム。

カードを手元で並べ替えるような動きに近く、要素を右へずらして空いた位置に入れる流れが特徴。

---

## 2. 基本の動き

挿入ソートでは、左から順に「いま見ている値」を取り出し、左側の整列済み区間に挿入する。

- 左端1要素は整列済みとみなす
- 次の要素を取り出す
- 左側にある要素と比較しながら、入る位置を探す
- 大きい値を右へずらす
- 空いた位置に現在値を入れる

これを末尾まで繰り返して整列させる。

---

## 3. 処理の流れ

処理の流れは次の通り。

1. 左端1要素を整列済み区間とみなす
2. 次の要素を現在値として保持する
3. 左側の整列済み区間を右から左へ比較する
4. 現在値より大きい要素を1つ右へずらす
5. 現在値が入る位置まで来たら、その位置に挿入する
6. 次の要素に進み、同じ処理を繰り返す

---

## 4. 計算量

- 時間計算量
  - 最悪: O(n^2)
  - 平均: O(n^2)
  - 最良: O(n)  
    すでにほぼ整列済みなら比較や移動が少なくて済む

- 空間計算量
  - O(1)

---

## 5. メリット / デメリット

### メリット

- 小さい配列では扱いやすい
- ほぼ整列済みの入力に比較的強い
- 現在値をどこへ入れるかという流れが理解しやすい

### デメリット

- 基本的には O(n^2) で遅い
- 配列サイズが大きくなるとシフト回数が増える
- 実務で大きな配列にそのまま使うことは少ない

---

## 6. Java実装のポイント

- `current` に現在値を保持している
- `j` を使って左側の整列済み区間を右から左へ見ている
- 現在値より大きい要素を右へシフトしている
- 最後に `current` を正しい位置へ挿入している
- 比較回数、シフト回数、参考実行時間を結果として返している

---

## 7. コアとなる処理・重要なコードの見どころ

最も重要なのは、現在値より大きい要素を右へずらし、空いた場所へ現在値を入れる部分。

```java
while (j >= 0) {
    comparisonCount++;

    if (values[j] > current) {
        values[j + 1] = values[j];
        shiftCount++;
        j--;
    } else {
        break;
    }
}

values[j + 1] = current;
```

ここで、

- 現在値 `current` より大きい要素を右へずらす
- 入る位置が見つかったら `current` を入れる

という、挿入ソートの中心処理を実現している。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic insertion-sort --trace
```

### 入力指定

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic insertion-sort --input 5,3,8,1,4 --trace
```

### 大きい入力

```bash
java -cp out io.github.seiya_matsuoka.sortingrecursion.App --topic insertion-sort --size 1000
```

実行時は、入力配列、シフトの流れ、挿入後の配列、比較回数、シフト回数、参考実行時間を確認できる。
